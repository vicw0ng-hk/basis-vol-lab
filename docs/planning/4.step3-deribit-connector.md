# Step 3: Deribit Connector

**Status**: ✅ Complete

## Goal

Implement an async Deribit collector that:

1. **Discovers active BTC and ETH instruments** (options, futures, perpetuals) via the public REST endpoint.
2. **Subscribes to ticker streams** for those instruments via the public WebSocket and emits parsed `TickerSnapshot`s.
3. Handles **heartbeats** and **reconnect/resubscribe** as part of the MVP, not polish (per the initial plan).

Public market data does **not** require a Deribit account or API key, so step 3 has no manual credential steps. (A Deribit account is still useful later — e.g. for testnet auth flows — but is not needed here.)

## Scope

In scope:

- `DeribitRestClient` — async REST client for `/public/get_instruments`.
- `DeribitWsClient` — async WebSocket client that:
  - connects to `wss://www.deribit.com/ws/api/v2`,
  - subscribes to `ticker.{instrument_name}.raw` channels,
  - sets a heartbeat (`public/set_heartbeat`) and answers `test_request`,
  - reconnects on disconnect and re-subscribes,
  - emits `TickerSnapshot`s through an async iterator.
- Pure parser functions (`parse_instrument`, `parse_ticker`) tested against fixture JSON — no network in tests.
- Mapping helpers: Deribit `instrument_name` → `Currency` / `AssetKind` / `expiry` / `strike`.
- Optional CLI entrypoint `python -m basis_connectors.deribit` for ad-hoc collection (wired to `mise run collect` later).

Out of scope (deferred):

- Authenticated/private channels.
- Backfilling historical volatility / index data (can be added in step 4 alongside Binance).
- Persisting to `MetadataStore` / `TimeSeriesStore` from inside the connector — the connector exposes domain objects; wiring is a separate concern.

## Design

```
basis_connectors/
  __init__.py
  deribit/
    __init__.py        # public exports
    parsers.py         # parse_instrument, parse_ticker (pure functions)
    rest.py            # DeribitRestClient (httpx)
    ws.py              # DeribitWsClient (websockets) with heartbeat + reconnect
```

### Parsers

- `parse_instrument(raw: dict) -> Instrument | None` — returns `None` for instruments outside our scope (non-BTC/ETH, expired, unsupported kind). Otherwise maps:
  - `instrument_name` → `symbol`
  - `base_currency` → `Currency`
  - `kind` (`"option" | "future"`) + `settlement_period == "perpetual"` → `AssetKind`
  - `expiration_timestamp` (ms) → `datetime` (UTC) for non-perpetuals
  - `strike` for options
  - `is_active` from response
- `parse_ticker(raw: dict) -> TickerSnapshot` — maps Deribit ticker payload to `TickerSnapshot`. Fields:
  - `timestamp` from `timestamp` (ms epoch).
  - `mark_price` from `mark_price`.
  - `bid` / `ask` from `best_bid_price` / `best_ask_price`.
  - `mark_iv` from `mark_iv` (options only; in % per Deribit, divide by 100 to get decimal).
  - `open_interest` from `open_interest`.
  - `funding_rate` from `funding_8h` for perpetuals (None otherwise).
  - `currency` / `asset_kind` derived from `instrument_name` via a small instrument-name parser.

### REST client

- `DeribitRestClient(base_url: str = "https://www.deribit.com/api/v2", client: httpx.AsyncClient | None = None)`.
- `await rest.get_instruments(currency, kind=None, expired=False) -> list[Instrument]`.
- Async context manager; closes the underlying `httpx.AsyncClient` if it owns it.

### WebSocket client

- `DeribitWsClient(url: str = "wss://www.deribit.com/ws/api/v2", heartbeat_interval: int = 30, ...)`.
- Public method `async iter_tickers(symbols: Iterable[str]) -> AsyncIterator[TickerSnapshot]`.
- Internally:
  - Tracks the live subscription set so reconnects can resubscribe.
  - Sends `public/set_heartbeat` after connect.
  - Replies to `test_request` heartbeats with `public/test`.
  - Reconnects with exponential backoff on connection close / network errors.
  - Yields each parsed ticker.

The WS layer is thin around the `websockets` library so the bulk of testing remains in the parsers.

## Notes from implementation

- Public clients **cannot** subscribe to `ticker.{instrument}.raw` — Deribit returns `raw_subscriptions_not_available_for_unauthorized` (error 13778). The default ticker interval is therefore `100ms`; set `ticker_interval="raw"` only with authentication (deferred).
- The WS client uses a queue + background receive task rather than yielding from inside `async with websockets.connect(...)`. The async-generator + websockets combination caused `aclose()` races on consumer break; the queue pattern shuts down cleanly via task cancellation.
- `mark_iv` is converted from Deribit's percent (`65.5`) to decimal (`0.655`) inside `parse_ticker`.
- `funding_rate` only populated for perpetuals (sourced from `funding_8h`).
