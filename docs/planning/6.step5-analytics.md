# Step 5: Analytics Package

**Status**: ✅ Complete

## Goal

Build the `basis_analytics` package: the math and signal layer that turns raw
ticker data from the Deribit and Binance connectors into the metrics the
dashboard will surface. Per the initial plan, this step uses **pandas** for
normalization/time-alignment, **NumPy broadcasting** for vectorized pricing,
**SciPy** for IV root solving and surface interpolation, and serializes
curated outputs as Parquet.

The interview-defining deliverable here is the **validation report** comparing
our own Black-76 IV/Greek calculations against Deribit's published `mark_iv`
and Greek fields, with explicit error bands by tenor.

## Why Black-76, not Black-Scholes-Merton

Deribit's `mark_iv` is quoted as the volatility under **Black-76 with the
underlying future as forward and zero discounting** (the option settles in
the underlying coin against the synthetic future at expiry). Using Black-76
with `F` = mark price of the matching future and `r = q = 0` is the only way
our IV reproduces Deribit's `mark_iv` to within numerical tolerance. The
inverse-quoting (option premium denominated in BTC/ETH) is a *display*
convention that doesn't change the IV solve.

This also keeps the pricer reusable: the same Black-76 engine prices any
forward-style option (e.g., a vanilla future option on a non-crypto venue).

## Modules in detail

### `pricing.py`

```python
def black76_price(F, K, T, sigma, *, is_call: bool, df: float = 1.0) -> ...
```

NumPy-vectorized: every argument can be a scalar or an array; broadcasting
follows NumPy rules. Returns the **undiscounted forward price × df**, where
`df` defaults to 1.0 (Deribit / cash-settled crypto convention).

Edge cases handled inline:
- `T <= 0` → intrinsic value `max(0, F-K)` / `max(0, K-F)`.
- `sigma <= 0` → intrinsic value (only if `T > 0`).

### `iv.py`

```python
def implied_vol_black76(price, F, K, T, *, is_call, df=1.0,
                        lo=1e-4, hi=5.0, tol=1e-8) -> float
```

Brent root finder via `scipy.optimize.brentq` on `pricer(σ) - price`.
Bracket `[1e-4, 5.0]` covers anything seen on crypto. Returns NaN when:
- input price violates no-arbitrage bounds, or
- the bracket doesn't change sign within tolerance.

Vectorized companion `implied_vol_black76_array(...)` loops in pure Python
(Brent isn't vectorizable) but is the right level of abstraction for
analytics callers.

### `greeks.py`

Analytic Black-76 Greeks for `delta, gamma, vega, theta, rho` returning a
`Greeks` dataclass. Vectorized (np.where for sign on calls vs puts).

We **do not** convert to Deribit's inverse-Greek convention in step 5:
Deribit's published Greeks are quoted in coin-terms (per 1 unit of
underlying), while ours are in standard forward terms. The validation
report uses a known coin-conversion factor `S * df` (where `S` is the
coin/USD reference) when comparing.

### `realized_vol.py`

```python
def close_to_close_rv(prices: pd.Series, *, window: str = "30D",
                      annualize: float = 365.0) -> pd.Series
def parkinson_rv(highs, lows, *, window, annualize=365.0) -> pd.Series
```

Both use `pandas.Series.rolling`. Index must be a `DatetimeIndex` (UTC).
Returns the **annualized** RV in decimal (0.55 = 55 %).

### `carry.py`

Existing `annualized_carry` is unchanged. Added:

- `annualized_funding(funding_rate_8h: pd.Series) -> pd.Series` —
  compounds 8h funding to an annualized number (`(1+r)**(3*365) - 1`,
  vectorized). Pandas Series in / out.
- `basis_curve(prices, expiries, *, spot, asof) -> pd.DataFrame` —
  vectorized annualized-carry across a futures curve.

### `surface.py`

```python
def atm_term_structure(snapshot_df) -> pd.DataFrame
def smile_interp(strikes, ivs, *, kind="cubic") -> Callable[[K], iv]
```

`atm_term_structure` picks the option closest to the forward at each
expiry and returns `(expiry, T, atm_iv)`. `smile_interp` returns a
`scipy.interpolate.PchipInterpolator` (monotone, no overshoot) wrapping
the strike → IV curve.

### `signals.py`

The three headline signals from the initial plan:

```python
def percentile_rank(s: pd.Series, *, window: str = "365D") -> pd.Series
def carry_vol_divergence(carry_pctile, iv_minus_rv_pctile) -> pd.Series
def skew_stress(rr_25d_pctile, oi_concentration_pctile) -> pd.Series
def regime_change_alert(carry_pct, skew_pct, oi_pct, *,
                        threshold: float = 0.8) -> pd.Series
```

All operate on pandas Series, all return Series. Each function is
intentionally a thin formula on top of percentile-ranked inputs — that's
the "non-magical, defensible in interview" framing.

### `validation.py`

```python
@dataclass
class IVValidationRow:
    instrument: str
    expiry: datetime
    strike: float
    is_call: bool
    deribit_mark_iv: float
    our_iv: float
    abs_err: float
    rel_err: float

def validate_iv(snapshots, futures_by_expiry) -> pd.DataFrame
```

Given a list of option `TickerSnapshot`s (with `mark_iv` set) and the
matching future mark price keyed by expiry, computes our IV from the
mark **price** and compares against `mark_iv`. Returns a DataFrame
suitable for `df.describe()` / per-tenor groupby.

The CLI `python -m basis_analytics.validate --currency BTC` prints a
per-tenor table: count, mean abs err, p95 abs err, max abs err.
