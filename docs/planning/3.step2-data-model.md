# Step 2: Local Data Model

**Status**: ✅ Complete

## Design decisions

Per the initial plan, the persistence layer uses **SQLite for metadata** (D1-compatible semantics) and **Parquet for time-series** (R2-compatible file layout). DuckDB is available as a dev-time tool for ad-hoc queries over Parquet but is not a runtime dependency.

### Why this split

- **SQLite/D1** handles relational metadata (instruments, collection runs) with low write volume. D1's free tier (100k writes/day, 5M reads/day) is more than sufficient for metadata.
- **Parquet/R2** handles time-series data efficiently — columnar compression, fast reads, and no egress fees on R2. Files are partitioned by `{venue}/{YYYY-MM-DD}/tickers.parquet` for simple date-range queries and cloud migration.

## What was implemented

### Contracts (extended)

Added to `packages/contracts`:
- `Instrument` — exchange-listed instrument with venue, symbol, currency, asset_kind, expiry, strike, is_active
- `CollectionRun` — metadata about a data-collection run (run_id, venue, timestamps, status, record count)

### Persistence package (`packages/persistence`)

Two modules:

**`metadata.py` — `MetadataStore`**
- SQLite-backed store for instruments and collection runs
- `upsert_instrument()` — insert or update (ON CONFLICT)
- `get_instruments()` — query with optional filters (venue, currency, asset_kind, active_only)
- `insert_run()` / `finish_run()` — track collection run lifecycle
- `get_runs()` — list runs, optionally filtered by venue
- All timestamps stored as ISO-8601 TEXT (D1 compatible)
- WAL mode + foreign keys enabled

**`timeseries.py` — `TimeSeriesStore`**
- Parquet-backed store for ticker snapshots
- `write_snapshots()` — write snapshots, auto-partitioned by venue and date; appends to existing files
- `read_snapshots()` — read all snapshots for a venue/date
- `list_dates()` — list available dates for a venue
- `read_date_range()` — read all snapshots within a date range
- Schema: `TICKER_SCHEMA` (PyArrow schema with microsecond UTC timestamps)
- File layout: `{base_dir}/{venue}/{YYYY-MM-DD}/tickers.parquet`

### Dependencies added

- `pyarrow>=20.0` added to `basis-persistence`
