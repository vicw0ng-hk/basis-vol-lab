# Step 4: Binance Connector

**Status**: ✅ Complete

## Goal

Implement an async Binance connector covering the data the dashboard needs from
the futures venue:

1. **Instrument discovery** for BTCUSDT/ETHUSDT perpetuals and quarterly futures
   (USD-margined contracts on `fapi.binance.com`).
2. **Live mark-price + funding** via the public `!markPrice@arr@1s` WebSocket
   stream.
3. **Historical backfills** for funding-rate history, basis, and open-interest
   statistics over the last ~30 days (the Binance public-data window).

Per the initial plan, Binance is our **futures/perpetual** venue: we use its
public mark-price/funding streams plus REST endpoints for funding history,
basis, and open interest. We do **not** use Binance for options.

## Scope

In scope:

- `BinanceRestClient` against `https://fapi.binance.com`:
  - `get_exchange_info() -> list[Instrument]` — filter to BTCUSDT/ETHUSDT
    perpetual + quarterly futures, mapped to `Instrument`.
  - `get_funding_rate_history(symbol, *, start, end, limit)` — paginated
    funding-rate history.
  - `get_basis(pair, contract_type, *, period, start, end, limit)` — basis
    series from `/futures/data/basis`.
  - `get_open_interest_hist(symbol, *, period, start, end, limit)` — OI
    history from `/futures/data/openInterestHist`.
- `BinanceWsClient` against `wss://fstream.binance.com`:
  - Subscribes to the combined `!markPrice@arr@1s` stream, which delivers
    mark price, index price, **estimated** next funding rate, and next funding
    time for **all** futures symbols every second.
  - Filters to instruments in our scope and emits `TickerSnapshot`s.
  - Sends pong frames in response to server pings; reconnects with
    exponential backoff (Binance closes connections after 24h).
- Pure parser functions tested against fixture JSON — no network in tests.
- CLI entrypoint `python -m basis_connectors.binance` for ad-hoc smoke runs;
  `mise run collect:binance` task.

Out of scope (deferred):

- COIN-margined futures, options.
- Spot or margin streams.
- Authenticated user-data streams.
- Persisting backfill results — the connector returns lists of domain
  objects; wiring to `TimeSeriesStore` is a separate step.
- Compositing both connectors into a single `mise run collect`.

## Design

```
                       # parse_funding_rate_row, parse_basis_row, parse_oi_row

```

### Symbol mapping

Binance uses uppercase concatenated symbols (`BTCUSDT`, `BTCUSDT_260626`).
- `contractType == "PERPETUAL"` → `AssetKind.PERPETUAL`, `expiry = None`.
- `contractType == "CURRENT_QUARTER"` / `"NEXT_QUARTER"` → `AssetKind.FUTURE`,
  expiry from `deliveryDate` (ms).
- `baseAsset` ∈ {"BTC", "ETH"} → `Currency`; everything else is filtered out.
- `status == "TRADING"` → `is_active = True`.

### Mark-price event

The `!markPrice@arr@1s` payload is an array of objects:

```json
{
  "e": "markPriceUpdate",
  "E": 1745971200000,
  "s": "BTCUSDT",
  "p": "76500.00",       // mark price
  "i": "76498.50",       // index price
  "P": "76450.00",       // estimated settle price (delivering futures)
  "r": "0.00012000",     // funding rate (perps only; "" for futures)
  "T": 1745988000000     // next funding time (perps only)
}
```

We map this to `TickerSnapshot` with `mark_price` from `p`, `funding_rate`
from `r` (if present), and **no** bid/ask/IV/OI on this stream — those
require separate streams or REST polls.

Stream URL: `wss://fstream.binance.com/market/stream?streams=!markPrice@arr@1s`.
Combined-stream payloads wrap the array under `{"stream": ..., "data": [...]}`.
(Pre-2026-04-23 the path was `/stream`; that legacy path now silently
delivers no `/market` channels.)

### Backfill endpoints

| Endpoint | Use |
|----------|-----|
| `GET /fapi/v1/exchangeInfo` | instrument discovery |
| `GET /fapi/v1/fundingRate` | historical funding (params: `symbol`, `startTime`, `endTime`, `limit≤1000`) |
| `GET /futures/data/basis` | basis series (params: `pair`, `contractType`, `period`, `limit≤500`) |
| `GET /futures/data/openInterestHist` | OI history (params: `symbol`, `period`, `limit≤500`) |

`period` is one of `5m`, `15m`, `30m`, `1h`, `2h`, `4h`, `6h`, `12h`, `1d`.

The parsers return small dataclasses:

- `FundingRateRow(symbol, funding_time, funding_rate, mark_price)`
- `BasisRow(pair, contract_type, timestamp, basis, basis_rate, futures_price, index_price)`
- `OpenInterestRow(symbol, timestamp, sum_open_interest, sum_open_interest_value)`

These live in `basis_connectors.binance.types` (connector-local; we'll lift
them into `basis_contracts` only if multiple packages need them).

### WebSocket client

- Default URL `wss://fstream.binance.com/market/stream?streams=!markPrice@arr@1s`.
- The `websockets` library handles control-frame ping/pong automatically; we
  rely on its built-in keepalive (defaults are appropriate for Binance).
- Reconnect with exponential backoff capped at 30 s.
- Filter the array payload by symbol set on each tick.

### REST client

- `BinanceRestClient(base_url="https://fapi.binance.com", client=None)`,
  async context manager.
- Public, unauthenticated endpoints only — no API key required.
- Returns parsed dataclasses; raises `httpx.HTTPStatusError` on non-2xx.

## Notes from implementation

- **The “geo-block” we initially diagnosed was misdiagnosed.** From Hong
  Kong, `wss://fstream.binance.com/stream?streams=!markPrice@arr@1s`
  consistently completed TLS + the WebSocket Upgrade handshake (server
  returned `101 Switching Protocols` in ~120 ms) but then never sent a
  single data frame. We confirmed the same silent behaviour through three
  different egresses (HK home network, JP commercial datacenter VPN, JP
  residential proxy) and a layer-by-layer probe (raw TCP, TLS, raw HTTP
  Upgrade all succeeded; only WebSocket frames were missing).

  **Actual root cause:** Binance's [Important WebSocket Change Notice](https://developers.binance.com/docs/derivatives/usds-margined-futures/websocket-market-streams/Important-WebSocket-Change-Notice).
  On **2026-04-23** Binance split futures WS traffic into
  `wss://fstream.binance.com/{public,market,private}` and decommissioned
  the legacy `/ws` and `/stream` paths for everything except `/public`
  channels. After that date the legacy paths still accept connections
  and SUBSCRIBE ACKs but stop pushing data for `/market` channels
  (`markPrice`, `aggTrade`, `kline`, `ticker`, `forceOrder`, etc.). This
  matches the user report in
  [`binance/binance-connector-python#526`](https://github.com/binance/binance-connector-python/issues/526)
  exactly. Switching the WS URL to `/market/stream?streams=...` fixed it
  immediately and the connector now streams live prod data from HK with
  no VPN.

  **Genuine jurisdictional restrictions still apply for deployment.**
  Binance Futures is not available in: United States, Canada,
  Netherlands, Cuba, Iran, North Korea, Crimea/Donetsk/Luhansk (per
  Binance's [List of Prohibited Countries](https://www.binance.com/en/about-legal/list-of-prohibited-countries)
  and futures-specific notes). For cloud deployment, **avoid AWS
  `us-east-1` / `us-west-2`**; any other region works. Tokyo
  (`ap-northeast-1`) and Singapore (`ap-southeast-1`) are the natural
  low-latency picks; Ireland (`eu-west-1`) and Frankfurt
  (`eu-central-1`) are equally fine.

  **Testnet `--testnet` flag retained** (CLI + `BinanceRestClient` /
  `BinanceWsClient` `base_url=`) since it's still useful for offline-ish
  smoke testing, but it's no longer needed for normal local dev. The
  `mise run collect:binance` task now points at prod.
- The `markPriceUpdate` event uses `r=""` (empty string) for delivery
  futures; the parser maps empty/missing `r` to `funding_rate=None`.
- `websockets`' built-in keepalive handles control-frame ping/pong; no
  custom keepalive is required for Binance.
