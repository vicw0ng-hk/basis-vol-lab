# Step 6: Product Shell — API, Web, Snapshot, Compose

**Status**: ✅ Complete

## Goal

Wrap the analytics + connectors built in steps 3–5 in a runnable, locally
demonstrable product:

1. A **snapshot script** that pulls one Deribit + Binance snapshot, persists
   tickers via `TimeSeriesStore`, and writes curated JSON artifacts that the
   front end consumes.
2. A **thin FastAPI service** (`apps/api`) that serves those JSON artifacts
   and exposes a `/api/refresh` endpoint to re-run the snapshot. Lambda-
   friendly via `Mangum`.
3. A **static React/Vite/Tailwind web shell** (`apps/web`) with four pages
   (Overview, Volatility, Carry, Signals) and a light/dark theme toggle.
4. A **Docker Compose** stack tying the two services together, sharing the
   `data/` volume so the API can re-run the snapshot and the web reads from
   the same artifacts. Tested with **OrbStack** as the Docker engine.

This delivers the working **MVP** referenced throughout `1.initial-plan.md`.

## Tech choices

### API: FastAPI

* `fastapi` + `uvicorn` for local dev; `Mangum` adapter so the same app drops
  into AWS Lambda later without code changes. Reads artifacts from disk and
  returns them as JSON. Includes permissive CORS for the local web origin.
* Endpoints (all GET unless noted):
  * `GET /healthz` — liveness.
  * `GET /api/meta` — when artifacts were generated, snapshot stats.
  * `GET /api/overview` — funding, basis, OI summaries.
  * `GET /api/vol` — ATM term structure + a smile node.
  * `GET /api/carry` — carry / funding / basis cards.
  * `GET /api/signals` — current carry-vol divergence, skew stress, regime alert.
  * `POST /api/refresh` — re-runs the snapshot in-process.

### Web: Vite + React + TypeScript + Tailwind v4 + Recharts

Rationale: the user asked for *simplistic but beautiful with popular tech*.

* **Vite + React 18 + TypeScript** — current default for static SPAs. Cloudflare
  Pages serves the `dist/` output directly.
* **Tailwind CSS v4** (via `@tailwindcss/vite`) — popular utility CSS, the v4
  Vite plugin needs zero config beyond a single `@import "tailwindcss"`.
* **`next-themes`-style toggle, native** — no Next.js dependency. We add a
  `class="dark"` toggle on `<html>` driven by a small `useTheme` hook backed
  by `localStorage` and the `prefers-color-scheme` media query. Tailwind v4
  picks it up via `@variant dark (.dark &)`.
* **Recharts** — the most popular React charting library; thin wrapper over
  D3 with declarative React API, sufficient for line/area/bar charts.
* **lucide-react** — clean icon set used by shadcn and many other modern UIs.

We deliberately **do not** install `shadcn/ui` via the registry CLI: that adds
generator state we don't need for four pages. Instead we hand-write a tiny
`Card`, `StatBlock`, `ThemeToggle` set using the same Tailwind tokens shadcn
uses (CSS variables on `:root` and `.dark`).

### Compose: minimal two-service stack

```yaml
services:
  api:
    build: { context: ., dockerfile: apps/api/Dockerfile }
    ports: ["8000:8000"]
    volumes: ["./data:/app/data"]
  web:
    build: { context: apps/web }
    ports: ["5173:80"]
    depends_on: [api]
```

OrbStack speaks the Docker socket exactly like Docker Desktop, so no special
configuration is needed; `docker compose up` works out of the box.

The API image is a multi-stage `python:3.14-slim` build that runs `uv sync
--package basis-api`. The web image builds with Node 24 and serves the
static `dist/` via `nginx:alpine`.

## Definition of done

* `mise run snapshot` writes a coherent set of JSON artifacts under
  `data/artifacts/` plus Parquet ticker files under `data/parquet/`.
* `mise run api` serves the artifacts on `http://localhost:8000`.
* `mise run web:dev` opens a SPA on `http://localhost:5173` showing all four
  pages with light/dark toggle, fed by the API.
* `docker compose up --build` starts both services on OrbStack and the same
  pages render against the containerised API.
* `mise run check` (format + lint + typecheck + tests) is green.

## Notes on OrbStack

OrbStack ships its own `docker` CLI shim that points at its socket; no
`DOCKER_HOST` override needed. The compose file uses **bind mounts** (not
named volumes) for `data/` so the host's snapshot artifacts are visible to
the containers — that round-trips correctly under OrbStack's bind-mount
implementation (it uses VirtIOFS, fast enough for our tiny JSON files).
